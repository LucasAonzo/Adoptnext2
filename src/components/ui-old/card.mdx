# Card Component

The Card component is designed with a focus on structure rather than styling, allowing for greater flexibility in design customization. This approach decouples the styling from the component structure, enabling a wider range of styling possibilities and design variations.

## Philosophy

The primary design philosophy behind this component is:

1. **Separation of concerns**: The component structure is separate from its visual styling
2. **Composability**: Components can be easily combined in different ways
3. **Flexibility**: Styling can be customized through className without modifying the component
4. **Minimal defaults**: Basic styling is provided but can be easily overridden

## Basic Usage

```tsx
import { Card, CardHeader, CardTitle, CardDescription, CardContent, CardFooter } from "@/components/ui/card"

// Basic usage
<Card>
  <CardHeader>
    <CardTitle>Card Title</CardTitle>
    <CardDescription>Card description</CardDescription>
  </CardHeader>
  <CardContent>Main content goes here</CardContent>
  <CardFooter>Footer content</CardFooter>
</Card>
```

## Customization

You can customize the appearance of any card component by providing a `className`:

```tsx
// Custom styling
<Card className="bg-primary text-white border-none">
  <CardHeader className="pb-2">
    <CardTitle className="text-xl">Custom Styled Card</CardTitle>
    <CardDescription className="text-white/80">With custom styling</CardDescription>
  </CardHeader>
  <CardContent className="pt-2">
    Content with custom padding
  </CardContent>
</Card>
```

## Creating Card Variants

Instead of building variants into the component itself, you can create custom variants using composition:

```tsx
// Primary card variant
function PrimaryCard({ className, ...props }) {
  return (
    <Card className={cn("bg-primary text-white", className)} {...props} />
  )
}

// Outlined card variant
function OutlinedCard({ className, ...props }) {
  return (
    <Card className={cn("bg-transparent border-2 border-primary", className)} {...props} />
  )
}
```

## Dynamic Heading Levels

The CardTitle component supports different heading levels through the `as` prop:

```tsx
<CardTitle as="h2">H2 Heading</CardTitle>
<CardTitle as="h4">H4 Heading</CardTitle>
```

## Common Patterns

### Interactive Cards

```tsx
<Card 
  className="transition-all hover:shadow-md cursor-pointer" 
  onClick={() => console.log('Card clicked')}
>
  <CardContent>
    This entire card is clickable
  </CardContent>
</Card>
```

### Cards with Links

```tsx
<Link href="/page">
  <Card className="h-full transition-all hover:-translate-y-1">
    <CardContent>
      Card content that links to another page
    </CardContent>
  </Card>
</Link>
```

### Card Grid Layout

```tsx
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <Card>
    <CardContent>Card 1</CardContent>
  </Card>
  <Card>
    <CardContent>Card 2</CardContent>
  </Card>
  <Card>
    <CardContent>Card 3</CardContent>
  </Card>
</div>
```

## Migration from Previous Version

If you were using the `inverted` prop in the previous version, you should now apply those styles directly:

```tsx
// Old way
<Card inverted>Content</Card>

// New way
<Card className="bg-primary text-white border-transparent">Content</Card>
```
